Welcome to ARTHA

ARTHA is a lightweight, student-first Java framework designed to make backend development as easy as solving a LeetCode problem. No XML, no complex annotations, just pure logic.

ðŸ’¡
Why ARTHA? It removes the boilerplate usually found in Spring Boot, allowing you to focus strictly on Data Structures and Algorithms logic within your endpoints.

Configuration

The heart of your application is the artha.json file. This single file manages your server settings, database connection, and dependencies.

{
    "port": 8080,
    "database": {
        "driver": "mysql",
        "host": "localhost",
        "port": 3306,
        "name": "artha_social",
        "username": "root",
        "password": "your_password"
    },
    "dependencies": {
        "hikari": "5.1.0",
        "jackson": "2.16.1"
    }
}

Key Settings

Key Description Example
port Server port number 8080
database.driver Database type (mysql, postgres, sqlite) "mysql"
dependencies Libraries to download automatically {"hikari": "5.1.0"}

Database Setup Guide

ARTHA supports multiple databases out of the box. Choose your flavor below.

ðŸ“˜
Sample Files: Copy these configurations directly into your artha.json.

Option 1: MySQL (Production Ready)

Best for standard web applications. Requires MySQL Server installed.

1

Configuration (artha.json)

"database": {
    "driver": "mysql",
    "host": "localhost",
    "port": 3306,
    "name": "artha_social",
    "username": "root",
    "password": "YOUR_PASSWORD"
}
2

Workbench Setup (schema.sql)

Open MySQL Workbench -> File -> New Query Tab. Paste and Run (Lightning Bolt icon):

CREATE DATABASE IF NOT EXISTS artha_social;
USE artha_social;

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(100) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL,
    password VARCHAR(255),
    full_name VARCHAR(255)
);

Option 2: PostgreSQL (Advanced)

Great for advanced data types and reliability.

1

Configuration (artha.json)

"database": {
    "driver": "postgresql",
    "host": "localhost",
    "port": 5432,
    "name": "artha_db",
    "username": "postgres",
    "password": "YOUR_PASSWORD"
}
2

Dependencies

Don't forget to add the driver in artha.json dependencies:

"dependencies": {
    "postgresql": "42.7.2", 
    "hikari": "5.1.0"
}

Option 3: SQLite (Easiest / Local)

Zero installation. The database is just a file in your project folder.

1

Configuration (artha.json)

"database": {
    "driver": "sqlite",
    "name": "data/app.db"   // File path relative to project
}
2

Initialization

You can run this Java code once to create tables (since there is no "Workbench" for SQLite by default):

Database.getInstance().execute(
    "CREATE TABLE IF NOT EXISTS users (" +
    "id INTEGER PRIMARY KEY AUTOINCREMENT, " +
    "username TEXT NOT NULL" +
    ")"
);

Simple ORM NEW

Stop writing raw SQL for everything! ARTHA's Simple ORM lets you treat database rows as Java Objects.

1. Define your Model

Use annotations to map JSON fields (CamelCase) to DB columns (SnakeCase).

public class User {
    private Integer id;

    @JsonProperty("full_name")  // Maps to DB column
    @JsonAlias("fullName")      // Maps from JSON input
    private String fullName;
}

2. Use in Controller

Magic happens here. Clean, readable code.

// 1. Auto-parse Request Body
User user = req.body(User.class);

// 2. Save directly to Database (Insert or Update)
db.table("users").save(user);

// 3. Update Response
return Map.of("status", "success", "id", user.getId());
